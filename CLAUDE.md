# Casinohub — アミューズメントカジノ向けトーナメント管理SaaS

## プロジェクト概要

アミューズメントカジノ（風営法の遊技場営業許可で運営される、換金なしのエンタメカジノ）向けのトーナメント管理SaaS。

### ターゲット
- トーナメントを定期開催しているポーカールーム
- アミューズメントカジノ店舗
- カジノバー・カジノイベント運営会社

### 解決する課題（メイン: トーナメント運営）
- トーナメントの参加者登録・チェックイン・レイトレジスト対応
- ブラインドストラクチャー管理・タイマー
- テーブル割当・テーブルブレイク時の再配置
- 順位集計・賞金/ポイント配分の自動計算
- リーダーボード・シーズン管理

### 収益モデル
- トーナメント参加者課金: ¥50/参加者〜
- チップ管理は将来無料機能として追加を検討

### 海外参考サービス
- LetsPoker（€1/プレイヤー/日、27カ国展開）
- Blind Valet（累計300万人以上）
- PokerAtlas TableCaptain（管理SaaS + 集客プラットフォーム一体型）
- 日本語対応 × 風営法準拠のサービスは存在しない → Casinohubの差別化ポイント

## プロジェクト構成

```
casinohub/
├── app/              ← 管理画面（Vite + React + TS + Tailwind v4）
│   └── src/
│       ├── contexts/AppContext.tsx   ← useReducer + localStorage 状態管理
│       ├── data/mockData.ts         ← モックデータ・初期値
│       ├── lib/supabase.ts          ← Supabaseクライアント
│       ├── lib/csvExport.ts         ← CSV出力ユーティリティ
│       ├── pages/AdminPage.tsx      ← メインレイアウト（4タブ構成）
│       ├── pages/DashboardTab.tsx   ← ダッシュボード（トーナメント中心）
│       ├── pages/TournamentTab.tsx  ← トーナメント管理（メイン機能）
│       ├── pages/ReportTab.tsx      ← 売上レポート（日次・月次）
│       ├── pages/SettingsTab.tsx    ← 設定（テーブル・スタッフ）
│       └── types/index.ts          ← 型定義
├── hp/               ← ホームページ（Next.js）
├── marketing/article/ ← オウンドメディア記事
├── 事業運営/          ← 事業運営タスク管理
├── images/           ← ロゴ・素材
└── 規約/             ← 法務文書
```

### 管理画面（app/）技術スタック
- **フレームワーク**: Vite 7 + React 19 + TypeScript
- **CSS**: Tailwind CSS v4（@tailwindcss/vite プラグイン）
- **バックエンド**: Supabase（現在はlocalStorage + モックデータで動作）
- **アイコン**: Lucide React
- **状態管理**: useReducer + localStorage（クロスタブ同期対応）
- **ポート**: 5176（dev server）

### 管理画面の機能
1. **ダッシュボード**: トーナメント状況、テーブル稼働状況、参加者数サマリー
2. **トーナメント**: 大会作成・参加者登録・ブラインドタイマー・テーブル割当・順位集計・賞金配分
3. **売上レポート**: 日次/月次集計
4. **設定**: テーブル・スタッフ管理

## フェーズ

競合SaaSがゼロなので、コンテンツで啓蒙する前にSaaS自体を出す。waiting list → β → 有料化の流れ。
コンテンツ（SEO記事）は並行で進めてドメインパワーを育てる。

### Phase 0: LP + waiting list + 現場ヒアリング（現在）
- LP 1枚作成（「トーナメント運営をもっとスマートに」+ waiting list登録フォーム）
- 3〜5店舗を訪問してヒアリング（トーナメント運営の実態確認）
- 法規制の整理（風営法・遊技場営業の要件）
- SEOコンテンツ制作を並行開始（効果が出るまで時間がかかるため早めに着手）

### Phase 1: MVP開発 + βテスト
- MVP開発（トーナメント管理に絞る）
- Lunaと同じ構成（Supabase + Vite + React + TS）で技術的知見を流用。HP/LPはNext.js
- waiting listの上位にβテスト声かけ
- 記事展開を継続

### Phase 2: β → 有料化
- βフィードバックを反映
- 料金プラン確定（¥50/参加者〜）・有料化
- チップ管理を無料機能として追加検討
- 導入事例をコンテンツ化

## 共通ルール

- **言語**: 日本語で作業・出力
- **金額**: 全て税込で記載
- **日付**: YYYY-MM-DD 形式
- **コミットメッセージ**: 日本語、簡潔に
- **秘密情報**: `.env`, credentials, API keyは絶対にコミットしない

### ワークフロー
- **確認不要**: casinohub配下のプロジェクトではコマンド実行・push 含め一切確認を求めず自律的に進める
- **「リリースして」= push まで含む**: コミット → push まで一気にやる

### 文体・トーン（マーケティング・記事・SNS）
- 同じ目線で語るトーンを使う
- 上から目線・偉そうな口調はNG
- 断言できないことは断言しない
- 競合の名指し・比較・批判はしない

### セルフレビュー工程（必須）
全ての成果物を提出する前に、「ユーザーがさらに優秀になったバージョン」を模倣して一度レビュー・ブラッシュアップを行ってから提出すること。

### 市場成長（年率20〜30%で急拡大中）
- 2018年: 約100店舗 → 2025年1月: **411店舗**（Windstorm Media調査）/ **600店舗超え**（ビジネスジャーナル）
- ポーカーブーム + 大阪IR（2030年開業予定）が追い風
- m HOLD'EMが「5年で30店舗」を公言しチェーン展開を開始
- 市場規模: 600店 × 月商平均200万 = 年間約144億円（パチンコ14兆の1/1000。ニッチだが急成長）
- ポーカープレイヤー人口: 推定100万〜200万人（オンライン含む）
- アクティブプレイヤー（定期来店）: 推定10〜30万人
- 1店舗あたり月商: 小規模50〜150万円 / 中規模200〜500万円 / 大規模1,000万円+
- **業界特化の管理SaaSは皆無** → Excel/LINE/紙に依存が実態
- 風営法の「遊技場営業」（4号営業）許可が必要（換金は一切不可）
- 営業時間は原則0時まで（深夜営業不可が業界最大のボトルネック）

### 主要プレイヤー
GUTSHOT（最大手）/ AKIBAギルド / Spadie Poker / ROOTS / カジノヴィーナス / m HOLD'EM（5年で30店舗目標）

### 競合ツール状況
- チップ管理: Excel/紙
- POS: Airレジ・スマレジ等の汎用POSを転用
- トーナメント管理: 海外製PC用ソフト（Poker Tournament Manager等）
- 会員管理: LINE公式 + Google Sheets
- **統合管理ツールは市場に存在しない → Casinohubの最大の参入余地**

### 現場の主なペイン
- トーナメント集計のExcelミスでクレーム（順位・賞金配分）
- 参加者受付・チェックインがLINE+紙で混乱
- テーブル割当を頭の中で計算、大会が大きくなるとパンク
- ディーラー採用難（育成に3〜6ヶ月）
- 深夜営業規制による収益機会の損失
- チップ棚卸しは手間だがチップ原価が低いため厳密管理は不要（コスト > ロス）

小規模店: 月5,000〜12,500円 / 中規模店: 月2〜4万円
