## 横断要件（全フェーズ共通）

### セキュリティ（OWASP Top 10準拠）
- SQLインジェクション防止: Supabase SDKパラメータバインディング必須（生SQL禁止）
- XSS防止: Reactデフォルト挙動（`dangerouslySetInnerHTML`禁止）
- 認証・認可: Supabase Auth + RLS（tenant_id）
- CSRF対策: SameSite Cookie + Supabase Auth
- 機密情報: `.env` / APIキーのコミット禁止

### 設計原則
- DDD/クリーンアーキテクチャ: UI → Application → Domain → Infrastructure
- ビジネスロジック（テーブル割当・賞金計算・ブラインド進行等）はドメイン層に集約

### パフォーマンス
- Web: Core Web Vitals（LCP < 2.5s, CLS < 0.1, INP < 200ms）
- API: Supabaseクエリ200ms以内
- タイマー: ブラインドタイマーの遅延は±1秒以内

### DB効率
- インデックス: `tenant_id`・`created_at`・外部キー・検索カラム
- N+1回避: JOINまたは一括取得
- SELECT指定: 必要カラムのみ
- ページネーション: 一覧系は件数制限

### セルフサーブUI設計（CS自動化）
1人で5プロダクト運用するため、問い合わせが発生しないUI設計を徹底する。
- **ツールチップ・インラインヘルプ**: 複雑な入力項目（ブラインドストラクチャー・賞金配分比率等）にはフィールド横に `?` アイコン + ツールチップで説明を表示
- **明確なエラーメッセージ**: 「エラーが発生しました」ではなく「参加者数は1人以上を入力してください」のように具体的な修正方法を提示
- **空状態ガイド**: データ0件時に「トーナメントを作成しましょう」等の次のアクションを案内
- **オンボーディング**: 初回ログイン時にステップバイステップのセットアップガイド（店舗設定→テーブル設定→ブラインド設定→トーナメント作成の順）
- **操作結果のフィードバック**: 全ての保存・削除・更新操作でトースト通知（成功/失敗を即座に伝える）
- **問い合わせは最終手段**: フォーム経由の非同期対応（inquiriesテーブル）のみ。チャットbot・電話サポートは設けない
